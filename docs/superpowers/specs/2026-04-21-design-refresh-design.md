# Design refresh ŌĆö Photoshop

## Goal

Refresh the visual design of the Photoshop web frontend (`web/`) to match
`new-design.html` (the reference mockup). Scope is colors, typography, spacing,
sizes, and two small layout tweaks. Functionality is unchanged. Alongside the
restyle, migrate CSS from a single 2231-line `styles.css` into a modular Sass
tree with a token-driven architecture so future visual changes stay cheap.

## Non-goals

- No changes to `web/app.js` logic beyond calling a new `renderIcons()` helper
  after DOM mutations that introduce icon stubs.
- No changes to DOM structure beyond replacing inline `<svg>` blocks with
  `<i data-lucide="ŌĆ”">` stubs and renaming CSS classes.
- No changes to element IDs, event handlers, keyboard shortcuts, Tauri
  integration, or file I/O.
- No deletion of features present in the current app but absent in the
  reference (home screen, new-document dialog, context menus, etc.). They are
  restyled, not removed.
- No build step added for anything other than Sass compilation.

## Success criteria

- `web/styles.css` is generated by Sass; no hand-authored CSS.
- `grep -n "#" web/styles/**/*.scss` returns hex values **only** inside
  `_tokens.scss`.
- `grep -nE '(__|_)' web/index.html web/styles/**/*.scss` is empty (no
  underscores in class names).
- `new-design.html` and the current editor view are visually aligned on:
  colors, typography, spacing, tool rail (2 columns), options bar height
  (32px), inspector, layers, status bar.
- All existing behaviors (open/save project, import image, draw, layer ops,
  filters, export, undo/redo, home screen, dialogs, context menus) continue
  working without regressions.
- Tauri build still produces an offline-functional desktop app (no CDN, no
  runtime Google Fonts fetch).

## Architecture

### CSS file structure

```
web/
  fonts/
    Montserrat-Medium.woff2
    Montserrat-SemiBold.woff2
  vendor/
    lucide.min.js
  icons.js                       ŌåÉ new; renderIcons() helper
  styles.css                     ŌåÉ compiled output (gitignored if desired)
  styles.css.map                 ŌåÉ sourcemap (gitignored)
  styles/
    index.scss                   ŌåÉ entry; @use every partial in cascade order
    _tokens.scss                 ŌåÉ design tokens (Sass vars + CSS custom props)
    _fonts.scss                  ŌåÉ @font-face for Montserrat
    _mixins.scss                 ŌåÉ shared mixins (icon-button, panel-surface, ŌĆ”)
    _reset.scss
    _layout.scss                 ŌåÉ app-shell grid, main grid, region sizing
    _utilities.scss              ŌåÉ .icon sizing, .visually-hidden
    components/
      _menubar.scss
      _tabbar.scss
      _options-bar.scss
      _tool-rail.scss
      _canvas.scss
      _contextual-bar.scss
      _inspector.scss
      _sections.scss
      _layers.scss
      _status-bar.scss
      _home-screen.scss

      _context-menu.scss
      _forms.scss
```

`web/index.html` continues to reference `./styles.css` ŌĆö the compiled Sass
output lands at that path so the HTML `<link>` does not change.

Every component partial is scoped to its own class prefix. No partial defines
hex values; all colors/sizes/spacing come from tokens in `_tokens.scss`.

### Naming convention

Flat kebab-case. No underscores. No BEM element/modifier separators. State
expressed via `.is-*` / `.has-*` classes toggled by JS.

```
.topbar                          .topbar-menu
.menu-link                       .menu-link.is-root   .menu-link.is-open
.home-screen                     .home-screen-sidebar .home-screen-nav-item
.tab                             .tab.is-active
.inspector-tab                   .inspector-tab.is-active
```

Migration treats HTML and SCSS as coupled: when a component's partial is
populated, the corresponding HTML class names change in the same commit.

### Token architecture (hybrid Sass + CSS custom properties)

Each token has three lines in `_tokens.scss`:

1. A private Sass variable (`$_foo`) holds the raw value.
2. `:root` emits `--foo: #{$_foo}` as a CSS custom property.
3. A public Sass variable (`$foo`) is set to `var(--foo)`.

Components only use the public `$foo`. Sass compiles `$foo` down to
`var(--foo)` in the output, so the runtime stylesheet uses CSS custom
properties throughout. Runtime theming (e.g., a future light mode or
"interface brightness" slider) can override tokens via `:root.theme-light
{ --bg-app: ŌĆ”; }` without recompilation. Dev ergonomics stay on the Sass
side: short `$accent`, IDE autocomplete, `@use` namespacing.

Sass-time math (loops, mixin internals) uses the private `$_foo`; runtime
math uses `calc($foo * 2)`, which compiles to `calc(var(--foo) * 2)`.

Sketch:

```scss
// _tokens.scss
$_accent:       #3b82c4;
$_accent-hover: #5397d5;
$_space-3:      6px;

:root {
  --accent:       #{$_accent};
  --accent-hover: #{$_accent-hover};
  --space-3:      #{$_space-3};
}

$accent:       var(--accent);
$accent-hover: var(--accent-hover);
$space-3:      var(--space-3);
```

## Fonts & icons

### Montserrat (self-hosted)

- Dev dep: `bun add -d @fontsource/montserrat` to obtain versioned `.woff2`
  files.
- Copy only the two weights actually used (500 Medium, 600 SemiBold) into
  `web/fonts/`. `@fontsource` is not shipped.
- `_fonts.scss` declares two `@font-face` rules with `font-display: swap`
  pointing at `../fonts/Montserrat-Medium.woff2` and
  `../fonts/Montserrat-SemiBold.woff2`.
- `body` inherits `$font-family`. System fallbacks (`system-ui`,
  `sans-serif`) cover the flash window before woff2 loads and the offline
  fallback if a font file is missing.

### Lucide (self-hosted, runtime swap)

- Dep: `bun add lucide`.
- Copy `node_modules/lucide/dist/umd/lucide.min.js` ŌåÆ `web/vendor/lucide.min.js`
  (one-time; committed or regenerated via a `bun run` script ŌĆö decide during
  implementation).
- Loaded via `<script src="./vendor/lucide.min.js"></script>` in
  `web/index.html` before `app.js`.
- A new `web/icons.js` exposes `window.renderIcons(root = document)`, which
  invokes `lucide.createIcons({ nameAttr: 'data-lucide', root })`. It runs
  once on `DOMContentLoaded` for the initial page.
- `app.js` calls `renderIcons(container)` after any DOM insertion that
  contains `<i data-lucide="ŌĆ”">` stubs: new layer rows, new tabs, opening
  dialogs, rendering the context menu, etc.

### Icon markup

```html
<i data-lucide="brush" class="icon"></i>
<i data-lucide="eye"   class="icon icon-sm"></i>
<i data-lucide="move"  class="icon icon-md"></i>
```

`_utilities.scss` defines `.icon` (14px, stroke-width 1.75), `.icon-sm`
(11px), and `.icon-md` (15px), all keyed to token values.

Before migration begins, produce an audit file mapping every existing inline
`<svg>` in `web/index.html` to a Lucide icon name. Most are direct matches
(`move`, `crop`, `type`, `pen-tool`, `brush`, `eraser`, `paintbrush`,
`pipette`, `paint-bucket`, `droplet`, `hand`, `zoom-in`, `eye`, `eye-off`,
`lock`, `plus`, `trash-2`, `chevron-down`, `copy`, `more-horizontal`, etc.).
Icons without a clean Lucide equivalent stay as inline SVG.

## Explicit layout changes

Only two structural changes beyond restyling:

1. **Tool rail: 2-column grid.** Rail width = `$w-tool-rail` (66px). Inner
   container uses `display: grid; grid-template-columns: 1fr 1fr; gap:
   $space-1`. Tool cells are 28├Ś26. Separators span both columns with
   `grid-column: 1 / -1`. The swatch block (foreground/background color) is
   centered across the grid.

2. **Options bar: 32px tall.** Height = `$h-optionsbar`. Inputs use
   `$h-control-sm` (20px). Buttons use `$h-control-md` (22px). Horizontal
   padding tightens to `$space-3`. Group separators become thin vertical
   `$border` lines.

Nothing else about the app shell's structure changes. Menubar, tabbar,
inspector, layers panel, canvas, status bar, home screen, dialogs, and
context menu keep their current HTML skeletons.

## Migration plan

To avoid a big-bang rewrite, the refactor proceeds in verifiable waypoints.
At each waypoint the app must load and all currently-working features must
continue working.

1. **Scaffold.** Create the full `web/styles/` tree with empty partials.
   Create `_tokens.scss` with the full token set. Create `_fonts.scss`,
   `_mixins.scss` (initially empty), `_reset.scss` (ported minimally from
   the original). Add Sass/concurrently/fontsource/lucide via Bun. Wire bun
   scripts. Compile to `web/styles.css`.

2. **Preserve existing styles temporarily.** Move the current contents of
   `web/styles.css` verbatim into a new `web/styles/_legacy.scss` partial,
   imported **first** by `index.scss` so later component partials cascade-
   win as they are populated. App renders identically. Sass now owns the
   compiled `styles.css`.

3. **Drop in fonts & icons infrastructure.** Place Montserrat woff2 files in
   `web/fonts/`. Place Lucide UMD in `web/vendor/`. Add `web/icons.js`.
   Update `index.html` with the two new `<script>` tags. App still renders
   identically (no icon stubs exist yet; Lucide has nothing to replace).

4. **Migrate components out of `_legacy.scss`, one at a time**, in
   risk-descending order:

   1. `_utilities`, `_reset`, `_layout`.
   2. Chrome regions: `_menubar`, `_tabbar`, `_options-bar`, `_status-bar`.
   3. `_tool-rail` (apply the 2-column layout change here).
   4. `_canvas`, `_contextual-bar`.
   5. `_inspector`, `_sections`, `_layers`.
   6. `_forms` (buttons, inputs, selects, mini-btns, segmented).
   7. `_home-screen`, `_dialogs`, `_context-menu`.

   Each step:
   - Moves the relevant rules out of `_legacy.scss`.
   - Rewrites them in the partial using tokens + mixins + new class names.
   - Updates HTML class names and replaces SVG blocks with Lucide stubs in
     the matching markup.
   - Updates any `app.js` selector references tied to renamed classes (rare;
     most `app.js` queries target IDs or `data-*` attrs, not classes).
   - Inserts `renderIcons()` calls at any `app.js` site that injects the
     migrated component's markup into the DOM.
   - Verifies the app loads, the migrated region looks correct against
     `new-design.html`, and nothing in not-yet-migrated regions regressed.

5. **Final sweep.**
   - Delete `_legacy.scss`.
   - `grep -n "#[0-9A-Fa-f]" web/styles/` must match only `_tokens.scss`.
   - `grep -nE '__|\b[a-z-]+_[a-z]' web/index.html web/styles/` must be
     empty.
   - Visual diff pass against `new-design.html`.
   - Smoke test in browser dev mode (`bun run dev`) and in Tauri mode
     (`bun run tauri:dev`).

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| A class rename breaks a JS selector and the regression is only caught visually | Before starting a component, grep `app.js` for every class name used by that component's current markup; any hit forces a JS update in the same commit. |
| Montserrat at the reference's small sizes (11ŌĆō12px) renders less legibly than Inter did | Verify during migration; if problematic, we can swap to Inter (cost: one token change) or adjust `-webkit-font-smoothing`. |
| Lucide icons render as empty `<i>` elements during a brief window before `lucide.createIcons()` fires | Load `lucide.min.js` and `icons.js` synchronously in `<head>` order (scripts without `defer`/`async`), call `renderIcons()` immediately after `DOMContentLoaded`. If flicker persists, set `<i data-lucide>` to `display: inline-block; width: 14px; height: 14px;` so layout is stable while empty. |
| `_legacy.scss` in transit accidentally shadows a new component's rules | Order `@use './legacy'` **first** in `index.scss` (so new component partials cascade-win on overlap), and delete each migrated section from it immediately. |
| Sass output path differs from what Tauri expects at package time | `web/styles.css` is the compiled output; Tauri packages `web/` and does not care about source. Document the `styles:build` prerequisite in README. |
| The compiled `styles.css` being gitignored causes a clean checkout to be unstyled until `bun install && bun run styles:build` | Add a `postinstall` script that runs `styles:build`. Document in README. |

## Open questions for implementation plan

- Is `web/styles.css` committed (simpler first-run experience) or gitignored
  (cleaner diffs)? Recommendation: gitignored + `postinstall` build hook.
- Should `web/vendor/lucide.min.js` be committed or re-copied from
  `node_modules` via a build script? Recommendation: committed ŌĆö it is a
  vendored dependency, small, and keeps the package.json simpler.

  tokens from the 7-step scale to a finer one if component needs exceed the
